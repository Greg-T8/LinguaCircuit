**SRS = Spaced Repetition System**

It’s a learning method that schedules reviews of information at **increasing time intervals**, based on how well you remember it.

---

## How it works (core idea)

Instead of reviewing everything repeatedly, SRS:

* Shows you a word
* You try to recall it
* You rate how well you remembered it
* The system schedules the next review:

  * **Hard → soon**
  * **Easy → much later**

---

## Example with a vocabulary word

Let’s say your word is *“obfuscate”*:

* Day 1 → Learn it
* Day 2 → Review
* Day 4 → Review
* Day 10 → Review
* Day 30 → Review

If you forget it at any point → the cycle resets to shorter intervals.

---

## Why it works

It’s based on the Ebbinghaus Forgetting Curve:

* You forget most new information quickly
* Reviewing **right before you forget** strengthens memory dramatically
* Over time, the memory becomes long-term

---

## What makes SRS different from normal studying

| Normal studying   | SRS              |
| ----------------- | ---------------- |
| Re-read notes     | Forced recall    |
| Fixed schedule    | Adaptive timing  |
| Recognition-based | Memory-based     |
| Inefficient       | Highly optimized |

---

## Where you see SRS used

* **Anki**
* **Quizlet (limited SRS)**
* **Duolingo (partial SRS)**
* Many language-learning and med-school tools

---

## Why it fits your use case

For vocabulary:

* Prevents “I saw this word once but forgot it”
* Forces you to **produce the word**, not just recognize it
* Can include:

  * definitions
  * sentence usage
  * roots/etymology

---

## Simple mental model

> Review **less often**, but at the **exact right time**

That’s what makes it powerful.

## 5. Stateful process pattern

This is the main pattern I would use for every lesson.

### Step 1: Select due items

PowerShell reads learner state and picks:

* overdue items
* recently missed items
* a few improving items
* optionally 1 or 2 new items

### Step 2: Build lesson packet

Generate a structured lesson packet for ChatGPT containing:

* learner profile
* today’s due items
* prior errors
* recent weak patterns
* lesson mode
* expected JSON output schema

### Step 3: Run lesson

ChatGPT conducts the lesson:

* asks recall questions
* tests sentence usage
* corrects nuance
* explains roots
* revisits misses

### Step 4: Capture structured result

At the end, ChatGPT returns something like:

* per-item assessment
* mistakes observed
* recommended status
* recommended next interval
* session summary
* newly inferred notes

### Step 5: Validate and persist

PowerShell:

* validates the JSON
* updates learner-state files
* writes a session-history log
* exports summaries if needed
* commits changes

That is the stateful loop.

## 7. Review algorithm

Start with a simple rule-based scheduler. Do not overcomplicate the first version.

Suggested initial intervals:

* missed: review again today or next day
* partial: 1 day
* correct but hesitant: 2 to 3 days
* clearly correct in context: 5 to 7 days
* strong repeated success: 14+ days

Track separate performance dimensions:

* definition recall
* sentence usage
* conversational use
* root/origin understanding

A word may be:

* definition strong
* usage weak
* nuance weak

That is more useful than a single raw score.

Later, you can evolve to:

* SM-2 style scheduling
* FSRS-style scheduling
* custom hybrid weights for conversation vs recall

## 8. Idioms need a slightly different model

Idioms are not just vocabulary entries.

Add fields like:

* literal gloss
* figurative meaning
* tone/register
* common contexts
* regionality
* fixed vs flexible wording
* anti-patterns or misuse examples

Example:

* `term`: "spill the beans"
* `literal_gloss`: "reveal contents"
* `figurative_meaning`: "reveal a secret"
* `register`: conversational
* `common_contexts`: secrets, surprise plans
* `misuse_notes`: not ideal in very formal writing

This lets ChatGPT teach idioms differently from single words.

## 12. How to handle conversational sessions

A common problem is that conversation is messy but state needs structure.

The fix:

* let the lesson itself be conversational
* end each session with a required structured recap block

So the session can feel natural, but the state update remains deterministic.

You can even require a final block like:

```text
BEGIN_SESSION_RESULT
{ ...json... }
END_SESSION_RESULT
```

Then PowerShell extracts that block and ignores the rest.
